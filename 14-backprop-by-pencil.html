The object: TN-1, reopened

📐 TN-1 (from page 6) A $2 \to 2\,(\tanh) \to 1\,(\text{sigmoid}) \to \text{BCE}$ network. Nine parameters. The frozen weights (constants.md §8): $$W_1 = \begin{bmatrix} 0.5 & -0.3 \\ 0.8 & 0.2 \end{bmatrix},\quad \mathbf b_1 = \begin{bmatrix} 0.1 \\ -0.1 \end{bmatrix},\quad W_2 = \begin{bmatrix} 0.6 & -0.9 \end{bmatrix},\quad b_2 = 0.2,\quad \eta = 0.1$$ Recall the invariant: $W$ is always $(d_{\text{out}}, d_{\text{in}})$. $W_1$ is $(2,2)$; $W_2$ is $(1,2)$. The target is $y = 1$.

Forward, the network is a chain of simple functions: a linear map, a $\tanh$, another linear map, a sigmoid, a loss. Backward, we want nine numbers — one per parameter — each answering "if I nudge you, how much does the loss change?" The chain rule says: to get any of them, multiply the local slopes along the path from that parameter to the loss. Backprop is just the chain rule ordered so no slope is ever computed twice.

The one quantity everything is built from: $\boldsymbol\delta$

Define, for every layer $\ell$,

$$\delta^{(\ell)}_i \;=\; \frac{\partial \mathcal{L}}{\partial z^{(\ell)}_i} \qquad\text{— "how much the loss changes per unit change in this pre-activation."}$$

This is the backprop workhorse. Once you have $\boldsymbol\delta$ for a layer, every parameter gradient in that layer is just $\boldsymbol\delta$ times a value you already cached on the way forward. That is the whole trick — and it is why one backward pass costs about the same as one forward pass.

📐 The four backprop equations

(BP1) Output error. $\;\boldsymbol\delta^{(L)} = \nabla_{\mathbf a}\mathcal{L} \odot \phi'(\mathbf z^{(L)})$. For BCE + sigmoid this collapses to the clean $\boldsymbol\delta^{(L)} = \hat y - y$.

(BP2) Recurse backward. $\;\boldsymbol\delta^{(\ell)} = \left(W^{(\ell+1)\top}\boldsymbol\delta^{(\ell+1)}\right)\odot \phi'(\mathbf z^{(\ell)})$.

(BP3) Weight gradient. $\;\dfrac{\partial\mathcal{L}}{\partial W^{(\ell)}} = \boldsymbol\delta^{(\ell)}\,\mathbf a^{(\ell-1)\top}$.

(BP4) Bias gradient. $\;\dfrac{\partial\mathcal{L}}{\partial \mathbf b^{(\ell)}} = \boldsymbol\delta^{(\ell)}$.

Shape check (BP2): $(d_\ell, d_{\ell+1})\times(d_{\ell+1},1) = (d_\ell,1)$, then $\odot\,(d_\ell,1)\to(d_\ell,1)$. ✓ And the universal law: the gradient has the same shape as the thing it differentiates — that is why param.grad has param.shape.

🔑 Why BP3 is the memory villain BP3 multiplies $\boldsymbol\delta^{(\ell)}$ by the cached activation $\mathbf a^{(\ell-1)}$. To take one backward step you must have kept every forward activation in memory. On a 9-parameter network that is nothing; on Qwen3-8B it is gigabytes — and it is exactly the term page 18 will call the activation cache. [THREAD:MEM] — hold this thought; the ledger reopens it.

🔑 The dead-unit lesson (the PREDICT answer) Because $a_{1,1} = 0$, the gradient $\partial\mathcal L/\partial W_{2,1} = 0$ — that weight cannot move this step, and nothing is broken: the unit is simply off for this input. Yet $\tanh'(0)=1$, so the gradient flows back through the unit undiminished — the row $\partial\mathcal L/\partial W_1[0] = [-0.3764, -0.7527]$ is fully alive. Two contrasting facts, one hand computation: a zero weight gradient and a healthy upstream gradient, side by side.

Why backprop is cheap

📐 One backward pass, not $P$ forward passes Every $\boldsymbol\delta$ is reused: the loss's sensitivity to a deep parameter travels through the same shared $\boldsymbol\delta$'s that every other parameter in that layer uses. So one backward pass, reusing the forward pass's cached values, yields all $P$ gradients at a cost of about one forward pass — $O(1)$ passes, not $O(P)$. The naive alternative, finite differences, would perturb each parameter and re-run the whole forward pass once per parameter: $P$ forward passes for the same nine numbers. At nine parameters that is a shrug; at eight billion it is the difference between training and not. [THREAD:CHAIN]

Three traps

⚠️ "A zero gradient means a dead or broken network." Here $\partial\mathcal L/\partial W_{2,1} = 0$ is correct and local: the unit $a_{1,1}$ is off for this input, so the weight reading its output has nothing to learn from this example. Flip to another input (Input 2) and it is nonzero again. A local zero is information, not a failure.
⚠️ "Backprop is a different algorithm from the chain rule." It is the chain rule — reverse mode — simply ordered so that each shared intermediate ($\boldsymbol\delta$) is computed once and reused, instead of re-derived down every path. Same math, smarter bookkeeping.
⚠️ "You must store the whole computation graph to get gradients." You store the forward activations (BP3's $\mathbf a^{(\ell-1)}$), not a symbolic graph. That storage is the training memory cost — the same activation cache the memory ledger charges you for on page 18.

🔬 Deep dive: reverse mode gives all $P$ gradients in one pass; forward mode would cost $P$

There are two ways to walk a chain of derivatives. Forward mode pushes a derivative $\partial\cdot/\partial\theta_j$ forward from one chosen input $\theta_j$ to the output — one pass yields the sensitivity of every output to one input. Reverse mode pulls $\partial\mathcal L/\partial\cdot$ backward from one chosen output — one pass yields the sensitivity of one output to every input.

Training has exactly one scalar output (the loss) and millions of inputs (the parameters). So reverse mode is the obvious fit: one backward pass delivers all $P$ gradients, where forward mode would need $P$ separate passes. That asymmetry — one loss, many parameters — is the entire reason every deep-learning framework computes gradients in reverse.
