Pretraining, Base vs Instruct

Next-token prediction is the whole objective — one loss, applied $10^{13}$ times, is where everything Qwen3-8B knows comes from. And it already explains why a "base" model won't stop talking while an "instruct" model answers and hands the microphone back.

⚠️ Builder disclosure — not captured Qwen3 logits spec-part5.md (line 73) calls for this demo to ship pre-recorded real Qwen3 logits. This build has no path to a live Qwen3-8B GPU forward pass, so the candidate logits the decode demo below runs on are hand-authored — chosen to reproduce the real, well-documented qualitative behavior (base rambles, instruct stops, template-wrapped base partially recovers), not captured from an actual forward pass. Everything downstream of those logits — softmax, argmax, stopping, cumulative log-prob, perplexity — is computed live, never precomputed or looked up. Pretraining installs everything the model knows

Pretraining is next-token prediction, run over roughly $10^{13}$ tokens of raw text. That is the entire mechanism — there is no separate "knowledge module." Every fact, every syntax rule, every association Qwen3-8B has, it acquired by getting slightly better, one gradient step at a time, at guessing the next token in text it had never seen before.

The objective, in frozen notation

Per-position cross-entropy, averaged over the batch and the sequence:

$$J(\theta) = -\frac{1}{BS}\sum_{b=1}^{B}\sum_{i=1}^{S} \log p_\theta\!\big(x_i \mid x_{<i}\big)$$

SymbolIsShape / rangeFrom
$J$the batch-averaged loss, in natsscalarnotation §2 — $J$, not $\mathcal L$, because it's already averaged
$\theta$every learnable parameter, flattened$(P,)$, $P=8{,}190{,}735{,}360$the trunk's parameter vector
$B$, $S$batch size, sequence lengthscalarsthe data loader
$x_i$the true token ID at position $i$integer in $[0,V)$the training sequence itself
$x_{<i}$every token before position $i$ — the prefix the model actually sees—the training sequence itself
$p_\theta(\cdot\mid x_{<i})$the model's softmax over the vocabulary, given that prefix$(V,)$the forward pass — page 44's $V=151{,}936$

Trunk callback: this is not a new loss. It's the exact cross-entropy-from-maximum-likelihood derivation from page 12, applied at every sequence position instead of once per example — Branch C of a derivation you already own, not a new fact (D-14). The only thing that's new here is scale: $V$ classes instead of a handful, and $S$ positions summed per example instead of one.

Translation note: this course writes $i$ for sequence position (notation §3). Papers and most codebases write $t$ for the same index — and in this course $t$ is reserved for the diffusion timestep, so every LLM-track page that indexes a sequence writes $i$ instead (notation §3.2).

Step-0 sanity check: perplexity

Perplexity is just the loss re-exponentiated: $\text{PPL} = e^{\mathcal L}$ — the same number, rescaled so "twice the loss" doesn't feel linear. It has a genuinely useful reading: PPL is the size of the vocabulary the model is, on average, as confused as if it were guessing uniformly among — put plainly, the number of equally-likely options it's effectively choosing from.

✍️ Worked — this is the predict's answer A freshly-initialized network has learned nothing yet, so its softmax over the $V=151{,}936$ possible next tokens is (approximately) uniform — every token equally likely, probability $1/V$ each. The cross-entropy loss of a correct answer under a uniform guess is $$\mathcal L = -\ln\!\left(\frac{1}{V}\right) = \ln V = \ln(151{,}936) = \mathbf{11.93}\ \text{nats}$$ [DER, constants §9.1], and perplexity at that same point is $$\text{PPL} = e^{\mathcal L} = e^{\ln V} = V = \mathbf{151{,}936}\quad\text{— exactly, by construction.}$$ Nothing was estimated here: an untrained softmax being uniform, and $e^{\ln V}=V$, are both exact. The step-0 loss is a hard floor every real Qwen3-8B training run starts just below.
⚠️ Not 11.76 You'll see 11.76 quoted as "the" random-init LLM loss in a lot of write-ups. That number is $\ln(128{,}256)$ — Llama-3's vocabulary size, not Qwen3's. Different tokenizer, different $V$, different floor. It's a retired number for this course (D-01); the two are easy to swap by accident because they differ only in the fourth significant figure.

Once training actually runs, a well-trained 8B model on general English web text typically settles around $\mathcal L\approx2.0$ nats, i.e. $\text{PPL}\approx7.4$ — but that number is corpus- and tokenizer-dependent, not a frozen constant, so it ships with a label: [EST, constants §9.1]. $\ln V$ itself, by contrast, is exact and worth keeping in your head: plot it as a horizontal "random-guess" line on every training-loss curve you meet from here on — a curve that starts near $\ln V$ and falls is doing exactly what it should.

⚠️ "Loss is comparable across models" Perplexity is only a fair comparison within one tokenizer. A different $V$ changes the random-guess floor ($\ln V$ itself), and a different compression ratio $\rho$ (page 44) changes how much "idea" one token even carries — so nats-per-token from two different tokenizers aren't measuring the same thing, even though both numbers are honestly in nats. People compare cross-tokenizer perplexity constantly in casual benchmarks; it's a category error every time.

[THREAD:Qwen3-8B beat: pretraining loss] — the same $V=151{,}936$ that set the tokenizer's shape on page 44 now sets the width of this loss's softmax, and $\ln V = 11.93$ joins the reflex-number set. It recurs as the exact loss SFT minimizes on page 48, with a mask bolted on, and as the metric that page 51 argues you should stop trusting once you care about answer quality rather than fluency.
