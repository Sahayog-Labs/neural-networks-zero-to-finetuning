Attention II: The √dhead Thermostat and Many Heads

Page 29 built the soft lookup and never justified the little $\sqrt{d_{\text{head}}}$ under the scores — the most hand-waved symbol in every explainer. Here we derive it, and find it is not numerical hygiene but a thermostat on the softmax's temperature: too cold and attention freezes into a gradient-free argmax; too hot and it melts into a uniform blur. Then we split one head into many and discover the heads are a reshape, not eight matrices. You set temperature in samplers already; this is the same knob, one layer down.

What is new: a head is a choice of directions to measure similarity along — its $W_Q,W_K$ pick which axes count. Many heads = many simultaneous opinions about "similar," run in parallel. 🔮 Predict first You feed a softmax logits that are 10× bigger than usual. Does it become (a) sharper / more one-hot, (b) flatter, or (c) unchanged — and what happens to the gradient flowing back through it? Commit before you scroll; you have set temperature in a sampler and felt half of this already.

The forward answer is (a): bigger logits make the softmax sharper, tending to a one-hot argmax. The half you may not have felt is the backward answer — as the softmax sharpens, its gradient dies. A one-hot vector is a step function in disguise, and step functions have no useful derivative. We will pin both halves to the decimal below, then let you drive a real attention layer off its detent and watch it happen.

🗺️ Translation note We write $d_{\text{head}}$ for the per-head width (Qwen3-8B: 128). Papers and configs write $d_k$, $d_v$, $d_h$, or head_dim. We write $i,j$ for query / key positions; papers write $t$, $m$, or $n$. The scale $\sqrt{d_{\text{head}}}$ is a fixed temperature $\tau$ — the same knob you turn in a sampler, here baked into the layer.

Where the $\sqrt{d_{\text{head}}}$ comes from

A score is a dot product of two $d_{\text{head}}$-dimensional vectors, $s = \mathbf{q}\cdot\mathbf{k} = \sum_{c=1}^{d_{\text{head}}} q_c k_c$. Add more dimensions and you are summing more terms — the score grows. Feed a softmax numbers that grow without bound and it stops being soft: it becomes a hard argmax, and a hard argmax has no gradient. The $\sqrt{d_{\text{head}}}$ is a thermostat that cancels that growth before it starts. Here is the exact constant, not a fudge.

Model $q_c, k_c$ as independent, mean-0, variance-1 (true at initialization, which is where training has to start learning). Then:

$$\mathbb{E}[s] = 0, \qquad \operatorname{Var}(s) = \sum_{c=1}^{d_{\text{head}}} \mathbb{E}[q_c^2]\,\mathbb{E}[k_c^2] = \sum_{c=1}^{d_{\text{head}}} 1\cdot 1 = d_{\text{head}} \;\;\Longrightarrow\;\; \operatorname{std}(s) = \sqrt{d_{\text{head}}}.$$

Not "roughly," not "for stability." A specific standard deviation, cancelled exactly. [THREAD: Qwen3-8B] With $d_{\text{head}}=128$: $\operatorname{std}(s)=\sqrt{128}=\mathbf{11.3137}$, so raw logits routinely land in $[-34,+34]$ (about $\pm 3\sigma$) before scaling.

What that does to the softmax — and to its gradient

Take the course's frozen three-logit example, $z=[2.0,\,1.0,\,0.1]$ (the same vector the cross-entropy, top-k, and temperature demos all use). Its softmax is a real blend:

✍️ Worked example — the blend, then the cliff $e^z = [7.389056,\,2.718282,\,1.105171]$, sum $=11.212509$, so $$\hat y = [\mathbf{0.659001},\ 0.242433,\ 0.098566].$$ A genuine mixture — every component gets weight, so gradient reaches all three. Now imagine these were raw attention scores at $d_{\text{head}}=128$ scale: multiply by $\sqrt{128}=11.3137$ to get $[22.63,\,11.31,\,1.13]$. Factor out the max and the softmax collapses: $$A \approx [\mathbf{0.99999},\ \sim 1.2\times10^{-5},\ \sim 10^{-10}]$$ — a one-hot vector in a trenchcoat. Same relative preference, catastrophically different softmax.

The forward pass survives either way — a one-hot output is still a perfectly good number. The damage is in the backward pass [THREAD: the chain rule]. The softmax's diagonal Jacobian is $\partial A_i/\partial s_i = A_i(1-A_i)$. Evaluate it for the winning key in each case:

Case$A_0$Jacobian $A_0(1-A_0)$Verdict
unscaled $z=[2.0,1.0,0.1]$$0.659001$$0.659001\times0.340999=\mathbf{0.2247}$healthy — gradient flows

In bfloat16 — 8 mantissa bits, relative resolution $2^{-8}\approx 0.004$ — a gradient of order $10^{-5}$ sitting against activations of order 1 simply rounds to zero. The update never happens.

🔑 Key idea Without the $\sqrt{d_{\text{head}}}$, attention does not get worse — it gets stuck. The softmax saturates, the Jacobian collapses, and $W_Q,W_K$ stop receiving gradient. The model can't learn what to look for, because looking has become a hard decision and hard decisions have no derivative.
🔬 Deep dive: why $\sqrt{d_{\text{head}}}$ is not quite enough after training — QK-Norm, and the temperature connection

The derivation assumed $q_c,k_c$ are mean-0, variance-1. That holds at initialization. After training, $Q$ and $K$ have whatever statistics the optimizer gave them — often with variance drift and large outlier channels — so a fixed $\sqrt{d_{\text{head}}}$ is no longer the exact normalizer. 2026 models patch this with QK-Norm: an RMSNorm applied to $\mathbf{q}$ and $\mathbf{k}$ before the dot product, re-imposing the unit-scale assumption every single step. Qwen3-8B ships q_norm and k_norm, each of size $d_{\text{head}}=128$ — you will see them in the state dict (256 parameters per block, a rounding error in the budget, load-bearing in the dynamics). It is a real design choice, not settled dogma: Cohere's Tiny Aya dropped QK-Norm and did fine.

And the scale is literally a temperature: $\tau=\sqrt{d_{\text{head}}}$. Dividing scores by a larger $\tau$ is exactly what turning down temperature in a sampler does to logits before the final softmax — flatten the distribution. The demo below lets you turn this knob directly.

Many heads — which is a reshape, not eight matrices

One head has one $W_Q,W_K$, so one notion of "relevant." Multi-head attention runs several relevance detectors on different learned subspaces at once — one head may track subject–verb agreement, another may just copy the previous token. [THREAD: the dot product] Each head is a choice of projection, exactly the page-27 point: which directions you measure similarity along changes what counts as similar. Many heads are many simultaneous choices.

$$\text{head}_h = \text{Attention}\!\big(XW_Q^{(h)\top},\, XW_K^{(h)\top},\, XW_V^{(h)\top}\big), \qquad \text{MHA}(X) = \text{Concat}(\text{head}_1,\dots,\text{head}_H)\,W_O^\top.$$

🔑 Key idea — the heads are a reshape You do not have $H$ separate matrices. There is one $(4096,4096)$ q_proj, and the "heads" appear when you call .view(B, S, 32, 128).transpose(1, 2). The math is 32 independent attentions; the code is one matmul and a view. When you first read modeling_qwen3.py and go looking for eight query matrices, this is why you won't find them.

Why does $W_O$ exist? Concatenation just stacks $H$ subspace-outputs that never interacted. $W_O$ mixes them back into one shared representation — and it is the head's write port onto the residual stream, a thread we pick up on page 34.

🔬 Deep dive: what heads actually learn — interpretable heads, polysemanticity, and attention sinks

Some heads are cleanly interpretable and reproduce across models: previous-token heads, positional heads, duplicate-token heads, and induction heads (the mechanistic substrate of in-context learning, a well-replicated result). But most heads are polysemantic — they do several unrelated things — and many can be pruned with little loss. The clean one-head-one-function pictures in papers are cherry-picked.

Attention sinks are real: many heads dump most of their weight onto token 0 / BOS regardless of content — a learned "no-op," because a softmax row must sum to 1 and sometimes a head has nothing it wants to attend to, so it parks the mass somewhere harmless. This is an emergent property of trained weights. The demo below uses seeded random projections, so its head-to-head differences are structural, not linguistic, and it will not show a sink — do not read grammar into it. What it will show, exactly and live, is the physics of the $\sqrt{d_{\text{head}}}$ thermostat.

Six traps

⚠️ Watch out

(a) "$\sqrt{d_{\text{head}}}$ stops numerical overflow." No — overflow is handled by the softmax's max-subtraction trick (page 9). The issue is gradient death via saturation: the forward pass is fine without the scale, the backward pass is dead. Fix by the Jacobian: $0.2247$ vs $1.2\times10^{-5}$.

(b) "Why $\sqrt{d_{\text{head}}}$ and not $d_{\text{head}}$?" Because we normalize a standard deviation, not a variance. Dividing by 128 would over-shrink by $11.31\times$, giving logits of std $0.088$ — a near-uniform softmax, the opposite failure. Both cliffs are on the demo.

(c) "$d_{\text{head}}$ is the model dimension." It is per head: $d=4096$ but $d_{\text{head}}=128$, so the scale is $\sqrt{128}=11.31$, not $\sqrt{4096}=64$. Using $d$ here is a common from-scratch bug.

(d) "More heads = more capacity." With $H\,d_{\text{head}}=d$, total attention params are identical regardless of $H$: $32\times128$ and $8\times512$ have the same count. It is a partition, not an addition. (The demo's code artifact computes both and asserts equality.)

(e) "Each head learns one linguistic function." Polysemanticity and prunability say otherwise; the clean pictures are cherry-picked (deep dive above).

(f) "Heads talk to each other inside the layer." They are completely independent until $W_O$ concatenates and mixes them. Inside the layer, head 3 has no idea head 7 exists.

Row you're inspecting — its weights over the 10 keys:

Three things to make happen, in order:

  1. Drag $\tau$ far below the detent. The heatmap snaps to a hard diagonal stripe, per-row entropy $\to 0$, effective keys $\to 1$, and the gradient to the top key goes red (down past $10^{-4}$) — the winner has saturated to 1, so $A_i(1-A_i)\to 0$ and no gradient reaches $W_Q,W_K$. You broke it by making it too confident.
  2. Drag $\tau$ far above. The map goes uniformly grey, entropy $\to\ln 10\approx 2.30$, effective keys $\to 10$, every row identical — the layer became a constant function. The gradient survives but points nowhere useful. You broke it by making it too humble; attention that attends to everything attends to nothing.
  3. Hit the detent. Structured, mid-entropy, healthy gradient. You have physically located the valley between two cliffs, and $\sqrt{d_{\text{head}}}$ is the thing that puts you in it. That resolves the prediction: bigger logits (smaller $\tau$) $\Rightarrow$ sharper forward, dead backward.

Switch heads at the detent: same sentence, genuinely different patterns, because each head measures similarity along different directions. Flip the causal mask and the upper triangle blacks out — that is page 32's whole story; here just note that the surviving weights regrow to keep each row summing to 1.

The code that proves the heads are a reshape

Try it — code/heads_are_a_reshape.py Takes a real Qwen3 q_proj shape, shows that .view(B,S,H,d_head).transpose(1,2) yields $H$ independent heads from one matrix; computes $P_{\text{attn}}$ for $(H{=}32, d_{\text{head}}{=}128)$ and $(H{=}8, d_{\text{head}}{=}512)$ and asserts they are equal; and prints the softmax Jacobian for scaled vs unscaled logits so you watch $0.2247$ become $\sim10^{-5}$.
Python
import torch
import torch.nn.functional as F

B, S, d, H, d_head = 1, 10, 4096, 32, 128
q_proj = torch.nn.Linear(d, H * d_head, bias=False)  # one matrix: (4096, 4096)
assert q_proj.weight.shape == (H * d_head, d)         # (out, in) — ALWAYS

x = torch.randn(B, S, d)
q = q_proj(x)                                       # (1, 10, 4096)
q_heads = q.view(B, S, H, d_head).transpose(1, 2)      # (1, 32, 10, 128) — 32 heads, no new params

# concatenating the heads is the INVERSE reshape — bit-for-bit, so they were a view:
q_concat = q_heads.transpose(1, 2).reshape(B, S, H * d_head)
assert torch.equal(q_concat, q)

# the scale's setpoint: std(q·k) at d_head=128 is exactly sqrt(128):
assert abs(d_head ** 0.5 - 11.3137) < 1e-4             # = 11.3137 (constants §5)

# (d) attention params depend on the projection WIDTHS, not the head count — 32×128 == 8×512:
def attn_params(d, H, d_head, H_kv):
    return 2 * d * d_head * (H + H_kv)         # q,o full width; k,v use H_kv heads
split_32 = attn_params(4096, 32, 128, 8)              # Qwen3-8B: 32 heads of 128, GQA H_kv=8
split_8  = attn_params(4096,  8, 512, 2)              # same widths, sliced coarser
assert split_32 == split_8 == 41_943_040             # a partition, not an addition

# the sqrt(d_head) thermostat, as a Jacobian:
z = torch.tensor([2.0, 1.0, 0.1])
for scale in (1.0, 128 ** 0.5):
    a = torch.softmax(z * scale, dim=-1)
    print(scale, a[0].item(), (a[0] * (1 - a[0])).item())  # 0.659, 0.2247  ->  ~1.0, ~1.2e-5
