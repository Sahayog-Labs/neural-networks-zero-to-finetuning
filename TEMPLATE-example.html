Math renders inline and display

A single neuron computes a weighted sum then an activation: inline, $z = \sum_i w_i x_i + b$, and $a = \sigma(z)$. The same thing set as a display equation:

$$a = \sigma\!\left(\sum_{i=1}^{n} w_i x_i + b\right), \qquad \sigma(z) = \frac{1}{1 + e^{-z}}$$

Callout boxes

🔑 Key idea Use .box.key for the one sentence a reader must leave with. A weight is just a number that scales an input; learning is nudging every such number a little in the direction that lowers the loss.
📐 Rule / definition Use .box.rule for a crisp definition or formula to memorize. The chain rule: $\frac{dL}{dw} = \frac{dL}{da}\cdot\frac{da}{dz}\cdot\frac{dz}{dw}$.
⚠️ Watch out Use .box.warn for the classic trap. Forgetting zero_grad() makes gradients accumulate across steps — with mini-batches the model diverges in tens of steps.
✍️ Worked example Use .box.worked for a read-only, pencil-and-paper derivation (green = "follow along", as opposed to amber = "go interact"). For $z = 0.5(0.6) + (-0.3)(-0.2) + 0.1 = 0.46$, then $a = \tanh(0.46) = 0.4301$.

🔮 Predict first A tanh unit's output is $a = 0.98$ — nearly saturated. Before you scroll, guess: is its local gradient $1 - a^2$ closer to 1, 0.5, or 0.04?

Resolving the predict right here, one sentence later: it's 0.04 — $1 - 0.98^2 = 1 - 0.9604 = 0.0396 \approx 0.04$. A nearly-saturated unit passes almost no gradient (the same fact the self-check quiz asks about below).

🔬 Deep dive: why the sigmoid derivative is $\sigma(1-\sigma)$

With $\sigma(z) = (1+e^{-z})^{-1}$, differentiate via the chain rule: $\sigma'(z) = \frac{e^{-z}}{(1+e^{-z})^2}$. Rewrite $e^{-z} = \frac{1-\sigma}{\sigma}\cdot\sigma$… and it collapses to the famous $\sigma'(z) = \sigma(z)\,(1-\sigma(z))$. This is why the gradient vanishes when a unit saturates: at $\sigma \to 0$ or $\sigma \to 1$, the product $\sigma(1-\sigma) \to 0$.
