One machine, many knobs

A neural network — however many billions of parameters it has — is one function. Numbers go in, numbers come out, and in between there are a great many small numbers you can turn: the knobs. When the network is running (inference), the knobs are frozen and only the input changes from call to call. When the network is learning (training), something flips: the input is fixed — it's your data, you don't get to change it — and the knobs are what moves. You turn them, a little at a time, to make one particular output number smaller: how wrong the machine currently is.

Say that flip out loud, because it is the single conceptual pivot the rest of this course hangs off of: training changes the knobs, not the input. Page 4 will give that sentence its symbols — what you differentiate with respect to is the knobs, with the input held fixed. For now just feel the shape of it: inference asks "what does this input produce?"; training asks "which nudge to the knobs makes the machine less wrong?" — a completely different question, aimed at a completely different set of things.

A neuron is a ruler and a bend

Zoom into the smallest piece. A single artificial neuron does two things, in order. First it acts like a ruler: it measures how much of the input points along a direction it has learned to care about. Second it bends that reading — squashes it through a soft S-shaped curve so the output always lands between "definitely no" and "definitely yes," with a smooth ramp in between rather than a hard jump.

Two separate facts about the ruler are worth pulling apart, because the demo above just showed you both. Its direction decides which way the dividing line faces — which side of the input space counts as "yes." Its length (how emphatically it measures) decides how abruptly the output snaps from "no" to "yes" as you cross the line — a short ruler gives a lazy, uncertain ramp; a long one gives a cliff. A third, separate knob — the offset — slides the whole line around without turning it at all. Three knobs, three distinct jobs; the demo let you feel the difference between all three.

A network is neurons feeding neurons

Layers are not different kinds of machine. Every layer performs the identical move — measure, then bend — and the output of one layer is simply the input to the next. "Deep" doesn't mean each layer learns a different job; it means the same simple move is repeated, stacked, many times over, so later layers can measure combinations of what earlier ones already noticed.

[THREAD:dot-product beat 0/…] — the "ruler" you just met has a name. It's called the dot product, and it is the single operation this entire course keeps coming back to: as a neuron's score, as an attention score, as an embedding similarity search, as the direction ComfyUI's CFG pushes an image toward. Page 3 gives it its name and its trigonometry — hang on to the word "ruler" until then.

Hold on to one more thing before you move on: later in this course, the machine itself never changes — only the question you ask it does. Predicting the next word and denoising an image both come out of the same measure-and-bend machine; the only thing that differs is the target it's trained toward. That's the fork this whole course eventually takes, and everything between here and there is building the one machine both sides need.

[THREAD:memory beat 0] — one more thing to file away and forget for now: those knobs also have to live somewhere, in your GPU's memory, every one of them. No numbers yet — page 18 is where you count them against your own box.

🔑 Key idea A neural network is one function with millions of knobs. Training is turning the knobs so the machine is less wrong. Everything else in this course is how you turn them fast enough to matter.
⚠️ Misconception: "the network looks up or stores the answer" It doesn't. A trained network is a deterministic function evaluation — the same weights, fed the same input, produce the same output, every single time. Any randomness you see in an LLM's replies or a diffusion model's images is added afterward, at sampling — it isn't the network changing its mind. Run the same prompt at temperature 0 twice and you get identical text; run it at temperature 0.8 and you get different text — but the weights never moved between those two runs. (The mechanism behind "temperature" gets its own page — 9.)
⚠️ Misconception: "layers are separate programs" They aren't. There is no internal boundary where one "program" hands off to the next — a network is one function, and a layer is just where we've drawn a bookkeeping line inside it. One layer's output feeding straight into the next is the entire mechanism; nothing is dispatched, looked up, or routed between them.
🔬 Deep dive: why one output and billions of inputs forces everything later to run backward

The loss is a funnel: it takes every one of a network's billions of knobs and collapses them down to a single number — how wrong the machine is right now. Asking "which direction, for each of billions of knobs, makes that one number smaller?" is a fundamentally different shape of question than "what does this input produce?" — and it is exactly that asymmetry, one output built from many inputs, that forces the gradient computation to run from the loss backward toward the knobs rather than forward alongside them. This is only a seed here; page 5 (the chain rule) picks it up properly, and Part II turns it into an algorithm you execute by hand.
