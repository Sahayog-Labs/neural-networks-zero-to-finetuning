The Neuron, XOR, and Why a Linear Stack Collapses

You have dragged a neuron (page 1) and named the ruler inside it (page 3). Now we write it down, stack a hundred of them, and prove the stack does nothing a single one couldn't — until you drop in one bend. That bend, not the depth, is the whole ballgame.

A neuron, precisely

Page 1 let you drag a neuron and watch it cut a plane in half; page 3 named the measurement it makes. Here is the same object with its symbols filled in — nothing new, just written down:

$$a=\phi(z),\qquad z=\mathbf w\cdot\mathbf x+b=\sum_{i=1}^{n} w_i x_i + b.$$

SymbolIsShape / unitsFrom
$\mathbf x$the input — one example$(n,1)$the data
$\mathbf w$the weight vector — a template the neuron matches against$(n,1)$a learned parameter
$b$bias — the threshold, negated: $z>0 \iff \mathbf w\cdot\mathbf x > -b$scalara learned parameter
$z$pre-activation — the match score, a dot productscalarcomputed above
$\phi$the activation (the "bend") — any nonlinearity—fixed function
$a$the neuron's output, $\phi(z)$scalarcomputed above

Read it geometrically — this is the reframing everything else rests on. The set $\mathbf w\cdot\mathbf x+b=0$ is a hyperplane: a line in 2-D, a plane in 3-D, a flat $(n{-}1)$-dimensional sheet in $n$-D. The vector $\mathbf w$ is its normal (it points straight across the sheet), and the sheet's distance from the origin is $|b|/\lVert\mathbf w\rVert$. So a neuron cuts space in half, and $\phi$ decides how sharply the output flips from "no" to "yes" across the cut. The weight vector is a template; $z$ is how strongly the input resembles it. [THREAD:dot-product beat 3] — $z$ is largest exactly when $\mathbf x$ points along $\mathbf w$ (callback to page 3's "how much of this is in that").

A layer is $m$ neurons at once

Give the layer $m$ neurons and stack their template rows into a matrix $W$. One matmul computes all $m$ match scores in a single shot — which is the only reason GPUs are worth pointing at this problem:

$$\mathbf h=\phi(W\mathbf x+\mathbf b),\qquad W\in\mathbb R^{m\times n},\ \ \mathbf b\in\mathbb R^{m},\ \ \mathbf h\in\mathbb R^{m}.$$

$W$one template row per output neuron — the course invariant, $(d_{\text{out}},d_{\text{in}})$$(m,n)$learned; pages 2, 7
$\mathbf b$one bias per output neuron$(m,1)$learned
$\mathbf h$the layer's output — $m$ activations$(m,1)$computed
$\phi$applied elementwise to all $m$ pre-activations—fixed

The shape bookkeeping is rigid, and it is the same ribbon you have been carrying since page 2:

shape ribbon
x        (n, 1)              # one input example
W        (m, n)              # (out, in) — ALWAYS
z = W x  (m, 1)              # m match scores, one per neuron
h = φ(z) (m, 1)              # same shape — φ is elementwise

Stack two of them with no bend — and watch it collapse

Here is the proof the predict is really about. Take two layers, but leave $\phi$ out — pure affine maps, $\mathbf h=W\mathbf x+\mathbf b$. Compose them:

$$\mathbf h_2=W_2(W_1\mathbf x+\mathbf b_1)+\mathbf b_2 =\underbrace{W_2 W_1}_{W'}\,\mathbf x+\underbrace{(W_2\mathbf b_1+\mathbf b_2)}_{\mathbf b'} = W'\mathbf x+\mathbf b'.$$

$W_1,\ \mathbf b_1$first layer's weights and biases$(m_1,n)$, $(m_1,1)$learned
$W_2,\ \mathbf b_2$second layer's weights and biases$(m_2,m_1)$, $(m_2,1)$learned
$W'=W_2W_1$the single equivalent weight matrix$(m_2,n)$collapses out
$\mathbf b'=W_2\mathbf b_1+\mathbf b_2$the single equivalent bias$(m_2,1)$collapses out

Two layers became one, exactly. Iterate the argument and a 100-layer linear network is a 1-layer linear network — same function, drawn with a straight boundary, no matter how deep or wide. The extra depth bought nothing except wasted parameters. Count them exactly (not "roughly"):

✍️ Worked — the wasted parameters, to the integer A two-layer linear net $784\to512\to10$ (the MNIST shape) holds $$\underbrace{784\times512}_{401{,}408}+\underbrace{512}_{\mathbf b_1}+\underbrace{512\times10}_{5{,}120}+\underbrace{10}_{\mathbf b_2}=\mathbf{407{,}050}\ \text{parameters.}$$ But it computes some $W'\mathbf x+\mathbf b'$ with $W'$ of shape $(10,784)$ — a single layer $784\to10$ worth $784\times10+10=\mathbf{7{,}850}$ parameters. So $1-\dfrac{7{,}850}{407{,}050}=0.9807=\mathbf{98\%}$ of the parameters are provably redundant. Insert one nonlinearity between the two layers and no such collapsing $W'$ exists — the composition is no longer affine. (Source: brief-foundations §10; arithmetic recomputed here.)
🔑 Key idea The nonlinearity contributes zero parameters and does all the work. Depth and width only matter because there is a bend between the layers; without it, they are elaborate ways of writing one line.

XOR by hand: build what a line provably cannot

The cleanest proof that the bend earns its keep is XOR. Four points — the corners of a square — with the diagonal pairs sharing a label:

$x_1$$x_2$XOR target
000
011
101
110

No single straight line separates the 1s from the 0s — try it, there is no such line. But two ReLU neurons and one output do it. Take the explicit weights below and verify all four inputs by hand:

$$h_1=\mathrm{ReLU}(x_1+x_2),\qquad h_2=\mathrm{ReLU}(x_1+x_2-1),\qquad y=h_1-2h_2.$$

✍️ Worked — all four corners, carried to a number
$(x_1,x_2)$$x_1{+}x_2$$h_1=\mathrm{ReLU}(\cdot)$$h_2=\mathrm{ReLU}(\cdot-1)$$y=h_1-2h_2$target

$(1,1)$22100 ✓
Output $[0,1,1,0]$ — exact. Notice $h_1$ and $h_2$ measure the same direction ($x_1+x_2$); they are the same line, shifted by 1. The only thing that tells them apart is the kink where ReLU clamps to zero. Four lines of arithmetic, and you built a function a linear model cannot represent at any depth. The ReLU did all of it.
📐 Where this leaves you You now need a bend, and you have proof — the collapse theorem on one side, XOR on the other. The next question is which bend, and why one of them (ReLU) quietly won the 2010s. That is page 11; the need is what carries you there.

Four traps

⚠️ "A deeper linear network is more expressive than a shallow one" The four-line proof above says no: $W_2W_1$ is just another matrix. Depth with no nonlinearity buys exactly one thing — a slower, larger way to compute the same affine map. Expressivity comes from the bend, not the layer count.
⚠️ "More neurons is the same as more layers" Width (neurons per layer) and depth (number of layers) are different axes. Width adds templates within one cut of space; depth composes cuts. The playground below lets you trade one against the other at roughly fixed parameter count and see they are not interchangeable.
⚠️ "The bias is a minor detail" Drop $b$ and every hyperplane $\mathbf w\cdot\mathbf x=0$ is forced through the origin — you lose the ability to slide a boundary off-center. Bias is essential in principle. (Modern LLMs like Qwen3 nonetheless set attention_bias: false and drop most biases, because RMSNorm makes them redundant in that specific architecture — that is a page-11/architecture point, not a contradiction of this one.)
⚠️ "Neurons are little brain cells" A 1943 marketing metaphor. An artificial neuron is a dot product and a squash — a template match with a bend. Say it once, then let the biology go; nothing in this course depends on it.
🔬 Deep dive: depth vs width, counted per parameter

Given roughly equal parameter budgets, a deep-narrow ReLU net and a shallow-wide one do not carve the plane equally. Each ReLU layer folds the input space along its unit boundaries; composing layers multiplies the number of distinct linear pieces, so a deeper net can produce many more linear regions per parameter than a wide-shallow one. That is why "just add width" is not a substitute for depth in practice — and it is exactly what the region counter in the playground below shows when you hold parameters fixed and move them from width into depth. This is an empirical tendency, not a theorem about which function is reachable; the universal-approximation caveats are page 11's.

Predict resolved: the answer is (b) — always a single straight line. With the activation set to identity, no depth and no width lets the boundary bend, because $\text{affine}\circ\text{affine}$ is affine (the collapse proof, felt in your hands). Flip to ReLU and the very same architecture becomes piecewise-linear and separates the moons.

Run this — code/xor_by_hand.py Hard-codes the ReLU XOR weights above, runs all four inputs through PyTorch, and prints [0, 1, 1, 0]. Then it "collapses" the net: two nn.Linear layers with no activation, trained on XOR — the loss floors and it never learns. Same lesson, in the framework, on CPU in under a second.
code/xor_by_hand.py
import torch, torch.nn as nn

X = torch.tensor([[0.,0.],[0.,1.],[1.,0.],[1.,1.]])
y = torch.tensor([0.,1.,1.,0.])

# --- XOR by hand: two ReLU neurons on the shared feature x1+x2 ---
h1 = torch.relu(X[:,0] + X[:,1])          # ReLU(x1 + x2)
h2 = torch.relu(X[:,0] + X[:,1] - 1)      # ReLU(x1 + x2 - 1)
out = h1 - 2*h2
print(out.tolist())                          # -> [0.0, 1.0, 1.0, 0.0]

# --- now collapse it: two Linear layers, NO activation ---
net = nn.Sequential(nn.Linear(2,8), nn.Linear(8,1))   # affine ∘ affine = affine
opt = torch.optim.Adam(net.parameters(), lr=0.1)
for _ in range(2000):
    loss = nn.functional.binary_cross_entropy_with_logits(net(X).squeeze(), y)
    opt.zero_grad(); loss.backward(); opt.step()
print(f"linear-net loss floors at {loss.item():.3f}")  # ~0.69 = random. It can't learn XOR.
