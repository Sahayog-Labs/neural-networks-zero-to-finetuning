Residual Connections and the Residual Stream

Two pages ago the RNN died of a product of shrinking Jacobians. A single "+ x" kills that death for good — and installs the most reusable mental model in the whole course: the residual stream, a shared bus that 36 committees write on in turn.

🧵 Thread [THREAD:chain-rule — the verb] · [THREAD:Qwen3-8B — the anchor]. This page reopens the chain rule you last met on page 28 as a product of Jacobians (the BPTT product that killed the RNN). Same verb, same failure — a product that decays — and the same fix the LSTM cell-state used: a near-identity Jacobian. What is new here is where the fix sits in a transformer, and the bus-shaped object it turns the model into. Qwen3-8B's real numbers ($L=36$, rms_norm_eps) anchor it, climaxing at the parameter count on page 36.

With no skip connection, how much of the layer-36 gradient survives back to layer 0: about 90%, 22%, or 2%? Commit before you scroll. The idea: compute only what to change

A plain layer is told: "here is $\mathbf{x}$; produce the next representation." A residual layer is told something humbler and far more useful: "here is $\mathbf{x}$; produce only what to change about it, and I'll add your change back on." In symbols the whole difference is one plus sign:

$$\mathbf{y} = \mathbf{x} + F(\mathbf{x};\theta).$$

The default behavior is now "do nothing": if $F$ outputs zero, the layer is the identity and the signal passes untouched. A 100-layer stack desperately needs each of its layers to be able to do nearly nothing — most refinements are small — and a plain layer has to learn the identity from scratch (hard), while a residual layer gets it for free (just push $F \to 0$).

But the forward-pass "it remembers the input" story is the secondary reason, and mistaking it for the main one is the classic error. The real mechanism is on the backward pass, and it is worth stating as a sentence you keep:

🔑 The gradient highway The + x gives the gradient a road from the loss straight back to layer 1 that does not pass through a single weight matrix. Nothing on that road can shrink it. Depth stops being fatal because there is always one clean path home.

The math: an $I$-term nothing can shrink

[THREAD:chain-rule] — the same verb as page 28, now with a gift term. Differentiate $\mathbf{y} = \mathbf{x} + F(\mathbf{x})$ with respect to its input:

$$\frac{\partial \mathbf{y}}{\partial \mathbf{x}} = I + \frac{\partial F}{\partial \mathbf{x}}.$$

The $I$ is the highway. Now chain it through all $L$ layers — this is exactly the BPTT product from page 28, but read what the identity does to it:

$$\frac{\partial \mathcal{L}}{\partial \mathbf{x}_0} = \frac{\partial \mathcal{L}}{\partial \mathbf{x}_L}\; \prod_{\ell=1}^{L}\left(I + \frac{\partial F_\ell}{\partial \mathbf{x}_{\ell-1}}\right).$$

Expand that product mentally. One term in the expansion is $I\cdot I\cdots I = I$ — an unattenuated path from the loss all the way to layer 0, untouched by any $\partial F$. Contrast page 28's RNN, whose product was $\prod \gamma$ with every $\gamma \le 1$: no identity term, so the product could only decay. The residual doesn't make the decaying terms disappear — it adds a term that can't decay, and at depth that term is the one that survives.

📐 Same problem, same fix, different decade The LSTM cell state (page 28) had a near-additive path with Jacobian $\approx \operatorname{diag}(\mathbf{f}_i)$, $\mathbf{f}_i \to 1$. A residual connection has Jacobian $I + \partial F/\partial\mathbf{x}$. These are the same idea: keep a route whose Jacobian is near the identity so the gradient has somewhere to flow that doesn't multiply it down. One fact, met twice, is one idea — not two.

The number that sells it — and resolves the predict

✍️ Worked example Take a healthy per-layer Jacobian norm $\gamma = 0.9$ and Qwen3-8B's depth $L = 36$ (num_hidden_layers = 36 [VP], constants §1.1).

$$\gamma^{L} = 0.9^{36} = 0.0225.$$

Only 2.25% of the layer-36 gradient reaches layer 0 — that is the predict's answer: about 2%. A "healthy" 0.9, compounded 36 times, still throws away 97.75% of the signal. Now switch the residual on. Each per-layer multiplier becomes $I + \partial F_\ell/\partial\mathbf{x}$; with a small, mean-zero learned update ($\|\partial F/\partial\mathbf{x}\|\approx 0.1$) the product stays $O(1)\approx \mathbf{1.0}$. The fold-improvement at $L=36$:

$$\frac{1}{0.9^{36}} = \frac{1}{0.0225} = \mathbf{44.4\times} \text{ more gradient.}$$

Push to a deeper 2026 model — GLM-5 is reported at $L = 78$ layers — and $0.9^{78} = 2.70\times10^{-4}$ vs $\approx 1$: about 3,700× (exactly $1/0.9^{78}$ when the update is negligible — set the demo's $\|\partial F/\partial\mathbf{x}\|$ slider to 0 to land on it). Residual connections are not an optimization. They are the thing that makes depth legal. The deeper you go, the more the residual is doing.

For calibration: the same base at $L=100$ gives $0.9^{100} = 2.7\times10^{-5}$ [DER, constants §9.3] — a hundred-layer plain stack passes essentially nothing. This is exactly the "amplifier" number from Part II, met again as an architecture verdict.

Try it Drive the highway yourself. Set the residual toggle off at $L=36$ and watch the surviving-gradient ratio dive to $0.0225$; toggle it on and it snaps back to $\approx 1$. Then drag $L$ out to 78 and watch the no-residual trace crater while the residual trace barely moves. Every point on both curves is computed live — $\gamma^\ell$ for the plain stack, the actual product $\prod(1+\partial F_\ell)$ for the residual one.

Install the model: the residual stream

Unroll the whole stack once and something clicks. Because every layer only adds, the state at the top is the input plus every contribution along the way:

$$\mathbf{x}_L = \mathbf{x}_0 + \sum_{\ell=1}^{L} F_\ell(\mathbf{x}_{\ell-1}).$$

SymbolIsShapeFrom
$\mathbf{x}_0$the embedded input — what gets written onto the bus first$(d,1)$the embedding, page 27
$F_\ell(\mathbf{x}_{\ell-1})$layer $\ell$'s contribution: it reads the bus, thinks, and adds$(d,1)$attention or FFN, page 35
$\mathbf{x}_L$the accumulated stream at the top, ready for the output head$(d,1)$this sum — the output head reads it
🔑 The residual stream The residual stream is a shared bus, $d = 4096$ wide, running the length of the model. Every layer reads from it, computes a contribution, and adds to it. Nothing is overwritten; everything accumulates. The model isn't a pipeline that transforms — it's a whiteboard 36 committees write on in sequence. This is the single most reusable mental model in the course; hold onto it.

Once you see the bus, a stack of otherwise-separate facts becomes one picture (each of these is cashed out later — this is the forward-reference list, not a detour):

  • Why $d$ is constant through the whole model. You can only add vectors of matching shape, so the bus width never changes. That "$d = 4096$ everywhere" you kept seeing is the bus, not a coincidence.
  • Why LoRA works. A LoRA adapter is just a small extra writer on the bus — a low-rank contribution added alongside a layer's — so it can steer behavior without rewriting the model (Part IV, page 40).
  • Why activation steering works. Add a fixed vector to the bus at some layer and you shift every downstream read — the operational face of page 27's linear-representation hypothesis.
  • Why $W_O$ matters. An attention head's output projection is its write port onto the bus (page 30); heads are independent until that matmul.
🔬 Deep dive: parallel blocks read and write the same bus at once

The bus model makes an otherwise-exotic architecture obvious. In a parallel block (Cohere's Tiny Aya, among others) the attention sublayer and the FFN sublayer both read the same $\mathbf{x}$ and both write their contribution to the stream, rather than the FFN reading the attention layer's output: $\mathbf{x}_{\ell} = \mathbf{x}_{\ell-1} + \text{Attn}(\text{norm}(\mathbf{x}_{\ell-1})) + \text{FFN}(\text{norm}(\mathbf{x}_{\ell-1}))$. On a bus this is trivially sensible — two committees writing the same round instead of taking turns — and it shortens the critical path. Nothing about the highway argument changes: both contributions still ride an $I$-term home.

Keep the highway clean: pre-norm, not post-norm

Normalization has to go somewhere in the block (page 20 said why you need it at all). The only question this page cares about is where relative to the + x, and it turns out to decide whether the highway stays clean.

PlacementBlockResidual pathVerdict
Post-LN (2017)$\mathbf{y} = \text{norm}(\mathbf{x} + F(\mathbf{x}))$a norm sits on the highway — no longer a clean $I$needs LR warmup; unstable past ~12 layers
Pre-LN (since ~2020)$\mathbf{y} = \mathbf{x} + F(\text{norm}(\mathbf{x}))$path stays exactly "$\mathbf{x}\ +\ $something" — highway intacttrains stably at 36 / 78 / 100+ layers

Read the two formulas against the highway argument: post-norm wraps a normalization around the sum, so the $I$-term picks up that norm's Jacobian and the road home is no longer unobstructed. Pre-norm normalizes only the input to $F$, leaving the outer $+\,\mathbf{x}$ pristine — the $I$ survives untouched. Every 2026 frontier model is pre-norm [V-sec]; this is a settled question, not a live tradeoff.

The norm itself: RMSNorm

And it is not LayerNorm anymore — it is RMSNorm, which throws away the mean-centering LayerNorm does and keeps only the rescaling:

$$\text{RMS}(\mathbf{x}) = \sqrt{\tfrac{1}{d}\textstyle\sum_{i=1}^{d} x_i^2 + \varepsilon}, \qquad \text{RMSNorm}(\mathbf{x}) = \mathbf{g} \odot \frac{\mathbf{x}}{\text{RMS}(\mathbf{x})}.$$

SymbolIsShapeFrom
$\mathbf{x}$the hidden vector being normalized (one bus reading)$(d,1)$the residual stream
$d$model width — the count we average overscalar = 4096config hidden_size [VP]
$\varepsilon$stability floor so a near-zero vector doesn't divide by zeroscalarconfig rms_norm_eps — see below
$\mathbf{g}$learned per-dimension gain (RMSNorm's only parameters — no bias)$(d,1)$trained
$\odot$elementwise product——

That $\varepsilon$ is not a mystery constant — it is a line in the config file you can open right now. For Qwen3-8B:

config.json
  "hidden_size": 4096,          # this is d — the bus width
  "num_hidden_layers": 36,     # this is L — 0.9^36 = 0.0225 without skips
  "rms_norm_eps": 1e-06,        # this is the ε in RMS(x)

So on Qwen3-8B, $\varepsilon = 1\times10^{-6}$ [VP], sitting inside the square root exactly where the formula puts it. No bias term appears because the learned gain $\mathbf{g}$ plus the downstream layers' own affine maps make one redundant — one fewer parameter per feature, at no measured cost.

🔬 Deep dive: QK-Norm is an RMSNorm, and depth-scaled variants

The q_norm / k_norm you'll meet assembling the block (page 30) are just RMSNorm applied to the per-head query and key vectors before the dot product — re-imposing the unit-variance that the $\sqrt{d_{\text{head}}}$ scale assumed. Same operation, different tensor. And 2026 labs tune the norm to depth: GLM-5 / Trinity use a depth-scaled RMSNorm whose effective gain accounts for how many contributions have already accumulated on the bus. All of it is variations on the one formula above — the bus stays the invariant.

Three traps

⚠️ "Residuals help the network remember the input." That forward-pass story is real but secondary. The load-bearing mechanism is the backward pass: the $I$-term nothing can shrink. If you only remember the "memory" story you will not predict that residuals matter more the deeper you go — but $0.9^{36} = 0.0225$ vs $\approx 1$ says they do.
⚠️ "Residuals let you make networks arbitrarily deep — deeper is better." They make deep networks trainable, not better. Returns diminish, and labs know it: GLM-5 cut depth from GLM-4.7's 92 layers to 78 while widening the bus ($d$: 5120 → 6144) [V-sec] — a 2026 lab deciding a previous generation had overshot on depth. Trainable is not the same as optimal.
⚠️ "x + F(x) breaks if $F$ changes the shape." Correct — and that is precisely why $d$ is held constant through the entire transformer: you cannot add mismatched vectors. CNNs, whose spatial dims change, need a $1\times1$ conv on the skip to match shapes; transformers never do, because the bus width is sacred.
