🔮 Predict first Classical theory says test error is U-shaped in model size: too small underfits, too big overfits. A modern LLM has billions of parameters and is fine-tuned on a few hundred to a few thousand examples — $P$ is enormously larger than $N$. By classical logic, that should be catastrophic overfitting. Is it? Before you read on: what do you think actually happens to test error as $P$ keeps growing past $N$?

Bias–variance — an identity, not a law

Fix a target $y$ and let $\hat f$ be whatever function your training procedure produces — it depends on which training set you happened to draw, so across many draws $\hat f$ is itself a random variable. Decompose the expected squared error into three non-negative pieces:

📐 The decomposition $$\mathbb E\!\left[(y-\hat f)^2\right] \;=\; \underbrace{\left(\mathbb E[\hat f]-f\right)^2}_{\text{Bias}^2} \;+\; \underbrace{\mathrm{Var}[\hat f]}_{\text{Variance}} \;+\; \underbrace{\sigma^2}_{\text{irreducible}}$$ This is algebra, not an empirical claim — expand $(y-\hat f)^2$ around $\mathbb E[\hat f]$ and every cross term vanishes in expectation. It holds for squared loss, always, for any model, any dataset size, any year. Nothing about it can go out of date.

✍️ Worked example — three datasets, one constant model Suppose the truth is a constant, $f(x)=1$, with irreducible noise variance $\sigma^2=0.01$. Three different training draws produce three different fitted constants: $\hat f_1=0.9$, $\hat f_2=1.1$, $\hat f_3=1.0$. $\mathbb E[\hat f]=(0.9+1.1+1.0)/3=1.0$, so $\mathrm{Bias}^2=(1.0-1.0)^2=0$ — this model class is unbiased here. $\mathrm{Var}[\hat f]=\big[(0.9{-}1.0)^2+(1.1{-}1.0)^2+(1.0{-}1.0)^2\big]/3=(0.01+0.01+0)/3=0.00667$. Total expected squared error $=0+0.00667+0.01=\mathbf{0.01667}$. Notice bias and variance are answering different questions: bias asks "is the average guess right?"; variance asks "how much does any one guess move if you re-run the experiment?" — a model can score zero on either one and still be wrong.

The classical prediction follows directly: as model complexity grows, bias falls (a richer model class can get closer to $f$ on average) but variance rises (a richer model class has more room to chase the noise in whatever training set it happened to see). Trade one against the other and test error traces a U. This was correct and useful for thirty years — for $P \ll N$, comfortably inside classical statistics' home turf.

Double descent — the U keeps going

🔑 Past the interpolation threshold, the curve comes back down Define the interpolation threshold as the point where the model has just enough capacity to fit the training set exactly, $P\approx N$. Classical theory predicts test error should keep rising past that point — more capacity, more overfitting, forever. It doesn't. Test error rises to a peak right at the threshold (variance is largest exactly where the fit is most brittle — barely enough capacity to interpolate, with almost no slack), and then descends again as $P$ keeps growing — often below the first, classical minimum. Not a U. A U followed by a second descent.

Resolving the PREDICT: a modern LLM sits far to the right of the threshold — $P$ in the billions against $N$ in the hundreds. Classical logic, extrapolated past where it was ever validated, says this should be catastrophic. It isn't, because the curve isn't a U past the threshold; it's already descended a second time by the time you get anywhere near GPT scale. Overparameterization buys you choice, not complexity — the intuition (Nakkiran et al., 2019): at $P\approx N$ there is essentially exactly one interpolating fit, and it is a brittle, high-variance one because there was no freedom in finding it. Past the threshold there are infinitely many functions that fit the training data exactly, and SGD's implicit bias (page 16 — the noise in the update path itself, not an explicit penalty) systematically favors the smoothest one among them. More parameters means more candidate interpolants to choose the smooth one from, not more freedom to overfit.

The phenomenon appears along three axes (Nakkiran et al., 2019 — "Deep Double Descent"): model-wise (sweep width or depth at fixed data, what the demo below does), epoch-wise (train one large model longer — test error can rise, peak, then fall again within a single training run), and sample-wise — the strange one worth calling out by name: near the threshold, adding more training data can make test error worse, because it can push a model that was comfortably over-parameterized right back onto the brittle threshold it had already grown past.

★ The honest part — watch the peak dissolve

⚠️ Do not present double descent as a settled law

(1) The peak largely disappears under proper regularization. Optimal early stopping or a well-tuned weight decay $\lambda$ substantially flattens it — the sharp spike at the threshold may be, in real part, an artifact of training an under-regularized model right at its most brittle capacity, not a separate law of nature bolted onto the bias–variance identity. The demo below turns $\lambda$ up and you watch this happen, live, on your own dataset — not a claim you're asked to take on faith.

(2) Parameter count is a poor complexity axis. Plot against a norm-based complexity measure instead (how large the fitted weights actually are, not how many of them exist) and the second descent can shrink or vanish — a lot of what looks like "more parameters helps" is really "more parameters makes it easier for SGD to find low-norm solutions."

(3) At LLM/ViT scale, published scaling curves are monotone power laws — smooth and monotonically improving, no double-descent bump where naive extrapolation says one "should" appear.

(4) What's actually dead is narrower than the headline. Only the claim that variance must rise monotonically with $P$ forever is false. The identity itself — Bias$^2$ + Variance + irreducible — is algebra and cannot be wrong.

Ship the honest framing: the classical U is real (small model, small data — often where you'll be). Double descent is real (a real reason big models exist and don't need to be exactly right-sized). Both are true in their regimes. The field does not have one clean theory that produces both from first principles. Hold both, and know which regime you're actually in.

★ The LoRA connection

🔑 LoRA's rank $r$ is the capacity knob — bias–variance in a 2026 costume LoRA restricts how much a fine-tune is allowed to move the model, and that restriction is a direct, literal capacity control: $r=4$ gives the update almost no room to move — high bias, it barely learns the new behavior, but it also can't memorize. $r=256$ fine-tuned on 20 images gives the update enormous freedom — low bias, but high variance: it doesn't just learn the concept, it memorizes the 20 photos and, in the process, forgets other things it knew. The classical U-curve is alive and living inside a LoRA config screen, and — the rule the trunk's LoRA/SVD pages (39–40) make explicit — the U's location is set by the dataset's information content, not by the base model's size. That's why the LLM track runs $r\approx 64\text{–}256$ (tens of thousands of varied examples, a lot to install) while the diffusion track runs $r\approx 8\text{–}32$ (fifteen to twenty images of one thing, almost nothing to install) — one rule, two regimes, not a contradiction. Not re-derived here; this page just hands you why it's the right rule when you get there. [THREAD:QWEN]

Four ways to misread this

⚠️ "Bigger models must overfit more." True only up to the interpolation threshold. Past it, that's the second descent — more capacity buys choice among interpolants, and SGD's implicit bias picks a smooth one. GPT-scale models live well past the threshold, where this intuition no longer applies.
⚠️ "The bias–variance tradeoff is obsolete — double descent replaced it." The decomposition is algebra; it cannot become obsolete. What's outdated is only the narrower claim that variance rises monotonically with $P$ forever. Honor the question mark — both pictures are true, in different regimes.
⚠️ "More data always helps." Almost always — except near the interpolation threshold, where sample-wise double descent means adding data can briefly make test error worse by pushing a comfortably over-parameterized model back onto the brittle threshold.
⚠️ "Double descent is a settled law I can rely on at any scale." The peak you'd rely on is largely a regularization artifact (watch it dissolve in the demo below), and at LLM/ViT scale the curves you'd actually see are monotone power laws with no bump. Don't bank on it; bank on the identity plus knowing your regime.

See it: the λ slider that dissolves double descent

Try it Leave $\lambda=0$ and watch: as hidden-layer width climbs, test loss falls, spikes hard right at the interpolation threshold (the dashed line, where model size $P$ crosses training-set size $N$), then comes back down — the classic U followed by a second descent. Now drag $\lambda$ up. Watch the peak flatten away — not because the axis changed, but because regularization was most of what built it.

Every point below is a real one-hidden-layer tanh network — the hidden layer is fixed random features (untrained, so it can't be accused of quietly doing the regularizing itself), and the output layer is solved exactly, in closed form, by ridge regression ($\lambda=0$ is the classical minimum-norm least-squares solution). That's deliberate, not a shortcut: it's the same construction Belkin et al. and Hastie et al. used to first pin the phenomenon down cleanly, it makes $P$ literally the count of parameters being fit to data (the $h$ read-out weights; the random hidden layer isn't "trained," so it isn't $P$), and it's fast enough to re-solve on every slider tick with no training-noise ambiguity about whether a spike is real or just an unlucky SGD run.

Fitting the sweep…
Try it on your box code/23_double_descent.py reproduces model-wise double descent on a small synthetic set — same construction as the browser demo (random hidden features, closed-form ridge read-out, so the spike is never in doubt) — sweeps weight decay to show the same peak flattening, and prints the regime test ($P$ vs $N$) for his actual fine-tune numbers — a few hundred examples against a multi-billion-parameter model.
Python
import torch

def ridge_fit_random_features(X_train, y_train, X_test, y_test, width, weight_decay, seed=0):
    # P = width = the ONLY trained parameters: the random hidden layer is fixed,
    # so it can't be quietly doing the regularizing itself (the browser demo's choice too).
    g = torch.Generator().manual_seed(seed)
    W1 = torch.randn(2, width, generator=g)
    b1 = torch.randn(width, generator=g)
    phi = lambda X: torch.tanh(X @ W1 + b1)
    Phi_train, Phi_test = phi(X_train), phi(X_test)
    # weight_decay=0 is the minimum-norm least-squares solution — the exact construction
    beta = torch.linalg.lstsq(
        Phi_train.T @ Phi_train + weight_decay * torch.eye(width), Phi_train.T @ y_train
    ).solution
    test_loss = ((Phi_test @ beta - y_test) ** 2).mean().item()
    return width, test_loss

# --widths sweeps P across the interpolation threshold P ≈ N_train;
# --wd 0.0 then --wd 0.05 reproduces the browser slider's "peak flattens" result.
