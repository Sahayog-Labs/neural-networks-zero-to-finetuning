The 2026 block, one line at a time

Do not read the finished block and back-fill reasons. Build it. Below is the exact structure every open frontier model shipped in 2026 shares — pre-norm, RMSNorm, GQA, QK-Norm, RoPE, SwiGLU, two residuals — written in the row-batched form with $W$ as $(d_{\text{out}}, d_{\text{in}})$. Every line is annotated with the earlier page that justifies it.

The block (Qwen3-style)
x = residual_stream                              # (B, S, 4096) — the bus, page 34

# --- attention sublayer ---
h = rms_norm_1(x)                                # pre-norm: normalize a COPY, keep +x clean (34)
q = h @ Wq.T                                     # Wq (4096, 4096)   → 32 query heads
k = h @ Wk.T ;  v = h @ Wv.T                     # Wk,Wv (1024, 4096) → only 8 KV heads: GQA (37)
q = q_norm(q) ;  k = k_norm(k)                   # QK-Norm: re-impose unit variance (30)
q, k = rope(q), rope(k)                          # RoPE on q,k only — never v (33). 0 params
q,k,v = to_heads(q,k,v) ; k,v = repeat_kv(k,v)   # (B, H=32, S, 128); tile 8 KV heads → 32
a = softmax(q @ k.T / sqrt(128) + causal) @ v    # causal mask = the LLM hinge (32). 0 params
out = a.reshape(B,S,4096) @ Wo.T                 # Wo (4096, 4096): the write port (30)
x = x + out                                      # highway #1 — the I-term nothing shrinks (34)

# --- feed-forward sublayer ---
h = rms_norm_2(x)                                # pre-norm again
ffn = Wdown @ (silu(Wgate @ h) * (Wup @ h))      # SwiGLU, THREE matrices (4096,12288)
x = x + ffn                                      # highway #2 → this x is the next block's input
📐 The six-line "why is this line here?" self-check This is the real test of whether pages 30/32/33/34/37 landed. Cover the answers; justify each from what you already know.
  1. rms_norm_1 before, not after → page 34: pre-norm keeps the residual highway x + … clean, so the $I$-term survives 36 layers.
  2. $W_K$ narrower than $W_Q$ ($1024$ vs $4096$ output) → page 37: GQA. 8 KV heads, not 32 — a 4× cut to the KV cache, and 37.5% off the attention block's weights.
  3. RoPE on q,k but not v → page 33: the rotation identity only pays off inside a dot product, and $\mathbf v$ is never dot-producted — it is averaged.
  4. + x twice → page 34: two sublayers, two clean highways, so gradient reaches layer 0 undiminished.
  5. Wo at the end → page 30: the heads run independently right up until this matmul, which is each head's write port back onto the bus.
  6. q_norm/k_norm → page 30 QK-Norm: it re-imposes the unit-variance the $\sqrt{d_{\text{head}}}$ scaling assumed, which raw projections drift away from.

The feed-forward network — bigger than you'd guess

Attention moves information between positions. The FFN thinks about it: same weights applied to every position independently, no communication between tokens. 4096 in, 4096 out, and a wide hidden layer in the middle. The 2017 version was two matrices with a ReLU: $\text{FFN}(\mathbf x)=W_2\,\text{ReLU}(W_1\mathbf x+\mathbf b_1)+\mathbf b_2$, with $d_{\text{ff}}=4d$. The 2026 version is SwiGLU — three matrices, no biases:

$$\text{FFN}(\mathbf x)=W_{\text{down}}\Big(\text{SiLU}(W_{\text{gate}}\mathbf x)\;\odot\;(W_{\text{up}}\mathbf x)\Big), \qquad \text{SiLU}(z)=z\,\sigma(z).$$

Why $d_{\text{ff}}=3d$, not $4d$? SwiGLU spends three matrices where the classic FFN spent two. To hold the parameter count fixed you shrink the hidden width by two-thirds: $\tfrac23\times 4d = \tfrac83 d \approx 2.67d$, and Qwen3 rounds up to exactly $3d$. So for the anchor $d_{\text{ff}} = 3\times4096 = \mathbf{12288}$ — the number sitting in intermediate_size. That is the answer to "why 12288 and not 16384": a parameter-budget correction for the third matrix. (The old "$d_{\text{ff}}=3.5d=14336$" you may have memorized was Llama-3.1's figure — retired here; the anchor is Qwen3-8B.)

⚠️ "No biases — did they forget them?" No: attention_bias: false [VP], and FLUX.2 drops bias throughout [VP] too. Once every sublayer is preceded by an RMSNorm carrying a learned gain $\mathbf g$, an additive bias is redundant — the gain already sets scale, and the residual stream carries any needed offset. Fewer parameters, one less thing to tune.

✍️ The count, per block

Attention. $W_Q = 4096\times4096 = 16{,}777{,}216$; $W_K=W_V=4096\times1024 = 4{,}194{,}304$ each (the GQA saving — 8 KV heads, not 32); $W_O = 4096\times4096 = 16{,}777{,}216$. Subtotal $= 2\cdot4096\cdot128\cdot(32{+}8) = \mathbf{41{,}943{,}040}$.

FFN. $W_{\text{gate}}=W_{\text{up}}=W_{\text{down}} = 4096\times12288 = 50{,}331{,}648$ each. Subtotal $= 3\cdot4096\cdot12288 = \mathbf{150{,}994{,}944}$.

Norms. $4096 + 4096 + 128 + 128 = \mathbf{8{,}448}$.

Block total $= 41{,}943{,}040 + 150{,}994{,}944 + 8{,}448 = \mathbf{192{,}946{,}432}$.

🔑 The parameter shock (your PREDICT, resolved) $$\frac{P_{\text{ffn}}}{P_{\text{block}}} = \frac{150{,}994{,}944}{192{,}946{,}432} = \mathbf{78.26\%}.$$ The FFN is 78% of the block — attention, the part every explainer dwells on, is barely a fifth. If you guessed 78%, you were right; almost everyone guesses attention. Attention gets all the attention; the FFN holds most of the weights. That is exactly why MoE (page 38) sparsifies the FFN — you sparsify the part that is 78% of the cost. (Careful with denominators: it is 78.26% of a block but 66.4% of the whole model, because the embedding tables dilute the whole-model share — both true, always say which. Never "~70%", the retired Llama figure.)
🔬 Deep dive: what the gate actually buys over a ReLU MLP

Split the two projections by role: $W_{\text{up}}\mathbf x$ is the content — a feature to maybe emit — and $\text{SiLU}(W_{\text{gate}}\mathbf x)$ is a per-dimension volume knob that lands mostly in $[0,1]$. Their elementwise product says "compute this feature, but only let it through if that other condition holds." That is a multiplicative interaction between two learned linear functions of the same input — something a ReLU MLP can only approximate by stacking depth. The gate is why SwiGLU beats a same-parameter ReLU FFN [M — the why is a practitioner reading; the config line is [VP]]. It is also why $\sigma$ with an explicit argument is legal here: this is the trunk, and $\sigma(\cdot)=$ the logistic sigmoid (SiLU is written by name in every config).

Encoder, decoder, encoder-decoder — and why decoder-only won

The block above is a decoder block: causal mask, next-token objective. It was not the only option. Three families competed, and one won generation outright — for reasons that are concrete, not aesthetic.

📐 Why decoder-only won — the honest four-part answer The single-cause story ("it's more elegant") is the misconception. There are four reasons, and the one that dominates is rarely stated first:
  1. Signal density. A causal LM gets a loss term from every token; masked LM from ~15%. That is $1/0.15 \approx \mathbf{6.7\times}$ more learning signal per forward FLOP. When compute is the bottleneck and the data is the whole internet, this dominates. [THREAD:chain-rule] — more tokens carrying a gradient means more of the backward pass does useful work.
  2. Task universality. Every task casts as text-completion; in-context learning falls out for free and was not designed in.
  3. Simplicity scales. One stack, one objective, one attention pattern — fewer knobs to get right at 1000-GPU scale.
  4. KV-cache economics. Causal masking means a token's $K,V$ never change once computed, so you cache them forever and decode incrementally (page 37). Bidirectional attention invalidates every previous token when you append one — no incremental decoding. Causal masking (page 32) is exactly what makes the cache sound.

Be honest about what was lost: bidirectional encoders are still better at representation — a token can depend on what comes after it — which is precisely why 2026 RAG uses dedicated bidirectional, contrastively-trained embedders, not decoder hidden states (page 27's anisotropy). Decoder-only won at generation, not at representation.

🔀 The fork beat — encoder-decoder didn't die, it moved A text-to-image model is an encoder-decoder. A text encoder — CLIP, T5-XXL, or Mistral-3 in FLUX.2 [VP] — produces the keys and values $K,V$; the U-Net or DiT is the decoder that cross-attends to them (page 31). The architecture you were told "lost" is the one you have been running in ComfyUI all along. Hold that thought: the two tracks of this course — LLM and diffusion — are the same block wired two ways, and page 64 grows them back together.
⚠️ Four traps (a) "Attention holds most of a block's parameters." The FFN does — 78.26%. Compute both subtotals: 41,943,040 vs 150,994,944.  (b) "$d_{\text{ff}}=4d$." Four concrete reasons, signal density first — not elegance.  (d) "Encoder-decoder is obsolete." It is what runs your ComfyUI text-to-image graph.

🔬 Deep dive: parallel blocks, and why 2026 labs still tune depth vs width

      
The residual-stream picture (page 34) makes two 2026 variants instantly legible. Parallel blocks
        (Cohere's Tiny Aya): attention and FFN both read the same normalized $\mathbf x$ and both write
        the stream, instead of the FFN reading attention's output — one fewer serial dependency, a hair faster, and
        the bus model shows why it barely changes the result (both are just writers on the same whiteboard).
        Depth-scaled RMSNorm (GLM-5, Trinity) rescales the norm by layer index to keep the stream's variance
        flat down a very deep stack. And depth itself is a live knob, not a solved one: GLM-5 cut depth
        92→78 while widening $d$ 5120→6144 versus GLM-4.7 [V-sec] — a lab deciding its previous model was too deep.
        None of this changes the block's skeleton; it is all knobs on the same six lines.
