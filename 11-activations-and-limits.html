The honest summary, first

In 2026: SwiGLU inside LLM/DiT feed-forward blocks; GELU or SiLU everywhere else; softmax at the output for classification; sigmoid only for gates and binary outputs. Sigmoid and tanh as hidden activations are dead. Everything below explains why — and why "dead" is not the same claim as "wrong" (page 6's TN-1 uses both, deliberately, and is fine).

The real reason ReLU won

⚠️ Not "it's cheaper to compute" An activation is a bandwidth-bound rounding error sitting next to the matmul it follows — speed was never the story. The real reason is gradient flow. [THREAD:chain-rule beat 0 reopened] — page 5's "gears" multiplied a chain of factors $\le 0.25$ and watched a 10-deep stack lose six orders of magnitude ($0.25^{10}=9.54\times10^{-7}$); page 4 first named $\sigma'(0)=0.25$ as sigmoid's ceiling. Cash that in now, at depth 32:
✍️ The idealized ceiling, cashed in $$\sigma'_{\max}=0.25 \quad\Longrightarrow\quad 0.25^{32}=\mathbf{5.4\times10^{-20}}$$ (constants.md §9.3 [DER].) This is the best case — every one of the 32 layers evaluated exactly at its steepest point, $z=0$. ReLU's local slope is exactly 1 on the active path, so its product never shrinks at all. That contrast — not FLOPs — is the whole story.

⚠️ A precision correction worth making out loud (constants.md §9.3 says this value sits "below fp32's smallest normal" — it does not). Float32's smallest normal number is $1.1754944\times10^{-38}$ (torch.finfo(torch.float32).tiny) — about $10^{18}\times$ smaller than $5.4\times10^{-20}$. The number is not below the representable floor; it is a perfectly ordinary fp32 float. What actually kills it is float32's precision, not its range: the machine epsilon is $1.19\times10^{-7}$ (constants.md §9.3), so adding an update of size $\eta\times5.4\times10^{-20}$ to any weight of order $10^{-2}$–$1$ rounds to exactly no change — the update is real, and it is also invisible to the hardware. "Practically zero," not "representably zero." The demo below plots both floors honestly, so you can see the gap for yourself.

Sigmoid's second sin: it is not zero-centered — every hidden unit's output is positive, so every next-layer weight sees same-signed gradients and the optimizer zigzags instead of stepping straight downhill. Tanh fixed exactly that ($\phi'_{\max}=1.0$, zero-centered) — and still lost, because $\phi'(x)\to0$ for $|x|>2$: it saturates just as hard, only slightly later. Zero-centering was never the bottleneck; saturation was.

ReLU's own sin — the dying unit: a ReLU whose pre-activation $z<0$ for every input in the dataset has gradient exactly $0$, forever — no update ever revives it. GELU ($x\,\Phi(x)$) and SiLU ($x\,\sigma(x)$) are the fix: smooth, and with a small but genuinely nonzero gradient on the negative side, so no unit is ever permanently dead. GELU's intuition is better than "a smooth ReLU": it multiplies $x$ by $\Phi(x)$, the probability a standard normal draw lands below $x$ — a soft, value-dependent stochastic gate, not a hard cutoff.

SwiGLU is a layer, not an activation

⚠️ The category error that breaks param counting $$\mathbf h = \mathrm{SiLU}(W_g\mathbf x)\odot(W_u\mathbf x)$$ has learned parameters, $W_g$ and $W_u$ — calling it "an activation" is a category error, and page 36's parameter count depends on you not making it. It also uses both $\odot$ (Hadamard) and matmul in one line — the cash-in of page 2's $\odot\ne@$ warning. "A wide-enough net can learn anything, so architecture doesn't matter." The universal approximation theorem is real — and it promises far less than that. Next section.

The MLP's honest limits

Cybenko (1989) and Hornik (1991): a wide-enough single-hidden-layer network with a non-polynomial activation approximates any continuous function on a compact set, to any accuracy. It is a real theorem, and it is the reason "neural networks can learn anything" became folklore. But read what it does not say:

  • It does not bound the width — "wide enough" could mean astronomically wide, unusable in practice.
  • It says nothing about whether gradient descent can find those parameters — existence is not reachability.
  • It promises nothing about generalization to data the network never saw.
  • It says nothing about depth — why stacking layers, empirically, so often beats making one layer wider (page 10's deep-vs-wide deepdive touched this; the theorem is silent on it entirely).
🔑 What the theorem actually buys you Universal approximation says a solution exists. It does not say you can afford it, train to it, or trust it off the training set. Everything from page 12 onward is about those four gaps — loss, optimization, generalization, and why depth is a different axis from width.

Genuinely open, not manufactured closure: nobody has a complete theory of why the wildly overparameterized networks this course builds toward generalize as well as they do. Say that plainly rather than paper over it.

🔬 Deep dive: the 2/3 rule — why a config says $d_{ff}=12288$

A vanilla two-matrix FFN ($\text{down}(\phi(\text{up}(x)))$) with the textbook width $d_{ff}=4d$ costs $2\times d\times 4d = 8d^2$ parameters. SwiGLU needs three matrices ($W_g,W_u,W_{down}$) instead of two. To match the vanilla FFN's parameter budget: $$3\,d\,d_{ff}^{\text{SwiGLU}} = 8d^2 \quad\Longrightarrow\quad d_{ff}^{\text{SwiGLU}} = \tfrac{8}{3}d \approx 2.667d.$$ Qwen3-8B uses $d_{ff}=12288=3d$ (constants.md §1.1) — closer to the honest $2.67d$ than Llama's $3.5d$ ($14336$); Qwen3 "spent less" on its FFN width per the same rule. This is the arithmetic behind a number you will see in every post-2023 LLM's config.json and never again have to wonder about.

Run it yourself

Run this — code/activations.py Tabulates $\phi'_{\max}$ for sigmoid/tanh/ReLU/GELU by autograd (not a table you memorized), prints $0.25^{32}=5.4\times10^{-20}$ against torch.finfo(torch.float32).tiny, then builds a real 32-layer sigmoid stack and a real 32-layer ReLU stack, backpropagates through each once, and prints the layer-1 gradient norm. CPU, under a second.
code/activations.py
import torch
import torch.nn as nn

torch.manual_seed(0)

# ---- 1. phi'_max for each activation, measured by autograd, not memorized ----
def phi_max(name):
    x = torch.linspace(-6, 6, 200001, requires_grad=True)
    if name == "sigmoid": y = torch.sigmoid(x)
    elif name == "tanh":    y = torch.tanh(x)
    elif name == "relu":    y = torch.relu(x)
    elif name == "gelu":    y = nn.functional.gelu(x)
    g = torch.autograd.grad(y.sum(), x)[0]
    return g.max().item()

for name in ["sigmoid", "tanh", "relu", "gelu"]:
    print(f"{name:8s} phi'_max = {phi_max(name):.4f}")

# ---- 2. the idealized ceiling, named against BOTH fp32 floors ----
ceiling = 0.25 ** 32
tiny    = torch.finfo(torch.float32).tiny     # smallest NORMAL, 1.1754944e-38
eps     = torch.finfo(torch.float32).eps      # precision floor, 1.1920929e-07
print(f"0.25**32       = {ceiling:.2e}")
print(f"fp32 min-normal = {tiny:.2e}  (ceiling is ABOVE this — still representable)")
print(f"fp32 epsilon    = {eps:.2e}  (ceiling is far BELOW this — an update this size vanishes on addition)")
assert ceiling > tiny            # representable...
assert ceiling < eps             # ...but invisible to any O(1) weight update

# ---- 3. a REAL 32-layer stack, one backward pass each ----
def deep_grad(act):
    x = torch.tensor(1.0, requires_grad=True)   # float32 by default
    h = x
    for _ in range(32):
        h = act(h)
    h.backward()
    return x.grad.item()

g_sigmoid = deep_grad(torch.sigmoid)
g_relu    = deep_grad(torch.relu)
print(f"32-deep sigmoid stack, layer-1 grad = {g_sigmoid:.3e}")
print(f"32-deep ReLU stack,    layer-1 grad = {g_relu:.3e}")
assert abs(g_sigmoid) < 1e-15, "sigmoid should be practically zero"
assert g_relu > 0.5, "ReLU's gradient should survive at ~1"
print("The myth said 'ReLU is faster.' Your own box says 'ReLU is the one whose gradient isn't zero.'")
