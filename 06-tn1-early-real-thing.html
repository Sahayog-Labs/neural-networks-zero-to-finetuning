🔮 Predict first TN-1's first hidden unit computes $z_{1,1} = 0.5(1) + (-0.3)(2) + 0.1$. Before you simplify — is that (a) exactly 0, (b) a small positive number, or (c) a small negative number? Commit to one, then read on.

Meet TN-1

TN-1 is deliberately tiny: two inputs, two hidden units with a $\tanh$ bend, one output unit with a sigmoid bend, and a loss on the end. Count the knobs — a $2\times 2$ weight matrix and a 2-vector bias into the hidden layer, a $1\times 2$ weight and a scalar bias into the output — and you get nine parameters. Small enough to run with a pencil; real enough that nothing about it is a lie.

📐 TN-1 — the frozen initial state A $2 \to 2\,(\tanh) \to 1\,(\text{sigmoid}) \to \text{BCE}$ network. Nine parameters, held fixed for this page (constants.md §8): $$W_1 = \begin{bmatrix} 0.5 & -0.3 \\ 0.8 & 0.2 \end{bmatrix},\quad \mathbf b_1 = \begin{bmatrix} 0.1 \\ -0.1 \end{bmatrix},\quad W_2 = \begin{bmatrix} 0.6 & -0.9 \end{bmatrix},\quad b_2 = 0.2,$$ $$\mathbf x = \begin{bmatrix} 1.0 \\ 2.0 \end{bmatrix},\qquad y = 1,\qquad \eta = 0.1.$$ The invariant from page 2 holds: $W$ is $(d_{\text{out}}, d_{\text{in}})$. $W_1$ is $(2,2)$; $W_2$ is $(1,2)$ — one output row, two input columns.

(Why $\tanh$ and sigmoid, when you may have heard they are outdated? Only because they are small, bounded, and hand-computable — perfect for a pencil. The full activation menu, and why these two are poor choices for hidden units in a big network, is page 11. The object transfers; the activations are incidental.)

Forward, by hand

The forward pass is a chain of simple functions, run left to right: a linear map, a $\tanh$, another linear map, a sigmoid, a loss. Here is the shape ribbon for the whole trip — every intermediate, with its shape, in the order the machine computes it:

Shape ribbon
x            (2,)                          # two inputs
W1           (2, 2)     b1  (2,)           # (out, in) — ALWAYS
z1 = W1 x + b1                (2,)         # two hidden pre-activations
a1 = tanh(z1)                 (2,)         # two hidden activations
W2           (1, 2)     b2  (1,)           # (out, in)
z2 = W2 a1 + b2               (1,)         # one output pre-activation
yhat = sigmoid(z2)            (1,)         # one probability
L = -ln(yhat)                 scalar       # BCE, since y = 1

Now put the numbers through it. Each pre-activation is a dot product plus a bias — [THREAD:dot-product], the atom from page 3, doing exactly the job it was named for:

✍️ TN-1 forward — every number Hidden pre-activations (row of $W_1$ dotted with $\mathbf x$, plus bias): $$z_{1,1} = 0.5(1) - 0.3(2) + 0.1 = \mathbf{0.0}, \qquad z_{1,2} = 0.8(1) + 0.2(2) - 0.1 = \mathbf{1.1}.$$ Hidden activations (the $\tanh$ bend): $$a_{1,1} = \tanh(0.0) = \mathbf{0.0}, \qquad a_{1,2} = \tanh(1.1) = \mathbf{0.8005}.$$ Output pre-activation (row of $W_2$ dotted with $\mathbf a_1$, plus bias): $$z_2 = 0.6(0.0) - 0.9(0.8005) + 0.2 = \mathbf{-0.5204}.$$ Output probability (the sigmoid bend) and the loss (BCE for a positive label, $y=1$): $$\hat y = \sigma(-0.5204) = \mathbf{0.3727}, \qquad \mathcal L = -\ln(\hat y) = -\ln(0.3727) = \mathbf{0.9869}\ \text{nats}.$$

Resolving the prediction: $z_{1,1} = 0.5 - 0.6 + 0.1$, which on paper is exactly 0 — answer (a). The first hidden unit is switched fully off for this input; $\tanh(0)=0$, so it passes nothing forward. (Hold that thought. When you run this in floating point on page 15, the machine's answer to "is it exactly zero?" turns out to be quietly, instructively no — but that is page 15's story, not this page's.)

⚠️ It's 0.9869, not 0.9870 You will find 0.9870 in more than one write-up of this network. It is wrong by one in the last place — it comes from rounding an intermediate to four digits and then taking its log. Carry full precision and the loss is $-\ln(0.3727472\ldots) = 0.9869$ (constants.md §8.1). Likewise $z_2 = -0.5204$, not $-0.5205$: same last-digit trap, caught on the real values. When a course promises "three representations, one number," the number has to actually be one number.

What is BCE doing on the end? For a positive example it is simply $-\ln$ of the probability the network assigned to the true label: assign $\hat y = 1$ and the loss is $0$; assign $\hat y \to 0$ and it blows up. TN-1 assigned $0.3727$ to a label that was true, so it pays $0.9869$ nats. That is all you need today; the full derivation of cross-entropy from maximum likelihood is page 12, and the softmax/cross-entropy family it belongs to is page 9.

Forward, in PyTorch

Here is the same network in about fifteen lines of PyTorch, runnable exactly as printed. It builds two linear layers, loads TN-1's nine frozen numbers into them, runs the forward pass on $[1.0, 2.0]$, and asserts the output matches your pencil to four decimals. There is deliberately no loss.backward() here — the backward pass is page 14's.

Python
import torch
import torch.nn as nn

torch.set_grad_enabled(False)              # forward only — no gradients on this page

lin1 = nn.Linear(2, 2)                  # weight shape (2, 2) = (out, in)
lin2 = nn.Linear(2, 1)                  # weight shape (1, 2)

lin1.weight[:] = torch.tensor([[0.5, -0.3], [0.8, 0.2]])
lin1.bias[:]   = torch.tensor([0.1, -0.1])
lin2.weight[:] = torch.tensor([[0.6, -0.9]])
lin2.bias[:]   = torch.tensor([0.2])

x    = torch.tensor([1.0, 2.0])
a1   = torch.tanh(lin1(x))               # two hidden units
yhat = torch.sigmoid(lin2(a1))         # one output probability
loss = -torch.log(yhat)                # BCE for the positive label y = 1

print(f"yhat = {yhat.item():.4f}")      # 0.3727
print(f"loss = {loss.item():.4f}")      # 0.9869
assert abs(yhat.item() - 0.3727) < 1e-4

🔑 Three representations, one number The pencil said $0.3727$. The live demo says $0.3727$. PyTorch says $0.3727$. Two traps

⚠️ "The framework does something I couldn't." It does not. Line for line, PyTorch's forward pass is the dot-products-and-bends you executed by hand. The only things it genuinely adds are speed and — starting page 14 — the backward pass, which is itself just the chain rule from page 5 run in reverse. Nothing on this page is beyond a pencil.
⚠️ "tanh and sigmoid are outdated, so this is a toy." They are chosen here because they are small, bounded, and hand-computable — not because they are what a real hidden layer uses (page 11 explains why they are poor there). What transfers is the object: a stack of dot products and bends ending in a loss. TN-1's nine parameters are the same shape of thing as the 8.19-billion-parameter model you will fine-tune later — only the count $P$ changes. [THREAD:memory beat 4]
